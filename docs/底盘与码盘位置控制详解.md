## 1. 底盘与码盘的关系

### 1.1 什么是码盘（编码器）

码盘（Encoder）是安装在电机轴或输出轴上的旋转传感器，通过光电或磁场原理将机械旋转运动转换为数字脉冲信号，从而精确测量：

- **旋转角度**（单圈 0~360° 或累积多圈角度）
- **旋转速度**（单位时间内的脉冲数 → RPM）
- **旋转方向**（顺时针 / 逆时针）

### 1.2 本项目使用的编码器型号

| 电机型号 | 编码器分辨率 | 编码器位置 | 减速比 | 用途 |
|----------|------------|-----------|--------|------|
| DJI GM6020 | 8192 脉冲/圈（CPR） | **输出轴**（减速后） | 1:1 | 舵轮转向电机 |
| DJI GM3508 | 8192 脉冲/圈（CPR） | **电机轴**（减速前） | 19:1 | 舵轮/全向轮驱动电机 |
| Unitree GO-M8010-6 | 内置多圈绝对编码 | 输出轴 | 6.33:1 | 机械臂关节电机 |

### 1.3 底盘结构与编码器分工

本项目底盘为 **4 轮舵轮结构**（Swerve Drive），每个角落一个舵轮单元，由两个电机协作：

```
┌─────────────────────────────────────────────────────────┐
│                       舵轮底盘俯视图                      │
│                                                         │
│         FL 左前                      FR 右前             │
│    [GM6020 转向]  ←→  底盘中心  ←→  [GM6020 转向]       │
│    [GM3508 驱动]                     [GM3508 驱动]       │
│                                                         │
│         RL 左后                      RR 右后             │
│    [GM6020 转向]                     [GM6020 转向]       │
│    [GM3508 驱动]                     [GM3508 驱动]       │
│                                                         │
│    wheel_base_x = 650mm (前后轮距)                       │
│    wheel_base_y = 620mm (左右轴距)                       │
│    wheel_radius = 55mm  (驱动轮半径)                     │
└─────────────────────────────────────────────────────────┘
```

- **GM6020（转向电机）**：编码器测量舵轮朝向角度（0~360°），用于位置闭环控制，使舵轮对准目标方向。
- **GM3508（驱动电机）**：编码器测量驱动轮转速（RPM），用于速度闭环控制，使底盘以目标速度前进。

## 2. 编码器读取与位置计算

### 2.1 CAN 反馈帧解析

DJI 电机通过 CAN 总线每隔约 1ms 上报一次状态帧，每帧 8 字节：

```
字节位置:  [0][1]    [2][3]   [4][5]     [6]
内容:    机械角度   转速RPM  电流值   温度°C
         (0-8191)  (int16)  (int16)   (uint8)
```

**代码示例**（`src/motor_control_ros2/src/dji_motor.cpp:57-60`）：

```cpp
// 解析 DJI 反馈帧 (Big Endian)
raw_angle_   = (static_cast<uint16_t>(data[0]) << 8) | data[1];  // 0~8191
raw_rpm_     = (static_cast<int16_t>(data[2]) << 8)  | data[3];  // RPM
raw_current_ = (static_cast<int16_t>(data[4]) << 8)  | data[5];  // 电流
raw_temp_    = data[6];                                            // 温度
```

### 2.2 GM6020 单圈角度换算（编码器在输出轴）

GM6020 的编码器直接安装在输出轴上，单圈映射关系为：

```
raw_angle ∈ [0, 8191]  →  物理角度 ∈ [0°, 360°]

转换公式：
  position_rad = (raw_angle / 8191.0) * 2π
  angle_deg    = (raw_angle / 8191.0) * 360.0
```

**代码实现**（`dji_motor.cpp:88-90`）：

```cpp
// GM6020: 编码器在输出轴，直接转换
position_ = (raw_angle_ / 8191.0) * 2.0 * M_PI;  // 弧度
```

### 2.3 GM3508 累积多圈计数（编码器在电机轴）

GM3508 的编码器安装在**减速前的电机轴**，存在两个特点：
1. 编码器只能感知单圈（0~8191），无法直接知道累积圈数；
2. 输出轴需经过 **19:1 减速比** 转换才能得到实际轮子位置。

因此需要**过零检测**来追踪累积圈数：

```
过零判断规则（编码器分辨率 8192，半圈 = 4096）：
  delta = 当前值 - 上次值
  if (delta > +4096):  编码器从 8191 → 0（逆时针过零），圈数 - 1
  if (delta < -4096):  编码器从 0 → 8191（顺时针过零），圈数 + 1
  否则：正常累积，圈数不变
```

**代码实现**（`dji_motor.cpp:62-90`）：

```cpp
if (!encoder_on_output_) {
    // GM3508: 编码器在电机轴，需要累积圈数
    if (!first_feedback_) {
        int32_t delta = static_cast<int32_t>(raw_angle_)
                      - static_cast<int32_t>(last_raw_angle_);
        
        // 过零检测
        if (delta > 4096) {
            encoder_rounds_--;       // 逆时针过零
        } else if (delta < -4096) {
            encoder_rounds_++;       // 顺时针过零
        }
    }
    
    last_raw_angle_  = raw_angle_;
    first_feedback_  = false;
    
    // 累积位置（电机轴弧度）= 累积圈数×2π + 当前圈内位置
    double rounds_position       = encoder_rounds_ * 2.0 * M_PI;
    double current_round_position = (raw_angle_ / 8191.0) * 2.0 * M_PI;
    position_ = rounds_position + current_round_position;
}
```

获取输出轴角度时除以减速比（`motor_base.hpp:162-168`）：

```cpp
double getOutputPosition() const {
    if (encoder_on_output_) {
        return position_;               // GM6020：直接返回
    } else {
        return position_ / gear_ratio_; // GM3508：除以减速比 19
    }
}
```

### 2.4 速度（RPM）换算

```
电机原始 RPM → 弧度/秒：
  velocity_rad_s = (raw_rpm / 60.0) * 2π
  
输出轴转速（GM3508）：
  output_rpm = raw_rpm / gear_ratio  →  raw_rpm / 19
  
驱动轮线速度（m/s）：
  v = output_rpm × wheel_circumference / 60
    = (raw_rpm / 19 / 60) × (2π × 0.055)
```

## 3. 脉冲数与实际距离/角度的换算标定

### 3.1 驱动轮（GM3508）：脉冲到地面距离

**理论换算公式**：

```
单个编码器脉冲对应地面距离：
  d_per_pulse = wheel_circumference / (encoder_resolution × gear_ratio)
              = (2π × R) / (8192 × 19)
              = (2π × 0.055) / 155648
              ≈ 2.22 × 10⁻⁶ m/pulse  （约 2.22 微米/脉冲）

累积脉冲到行进距离：
  distance = N_pulses × d_per_pulse
```

**实际标定步骤**：

```bash
# 1. 启动控制节点并记录电机状态
ros2 run motor_control_ros2 motor_control_node &
ros2 topic echo /dji_motor_states | grep -A5 "DJI3508_1" > before.txt

# 2. 底盘沿直线行驶已知距离（如 1.000 m），并停止

# 3. 再次记录电机状态
ros2 topic echo /dji_motor_states | grep -A5 "DJI3508_1" > after.txt

# 4. 计算实际脉冲/距离比（encoder_rounds × 8192 + 当前单圈值）
#    理论值：行驶 1m ≈ 155648 个电机轴脉冲
#    请将下方 pulses 替换为实测脉冲差值
python3 -c "
dist_actual = 1.000   # 实际行驶距离（m）
pulses = 155648       # 替换为实际读取到的脉冲差值（电机轴，理论约 155648）
print(f'实测：每脉冲对应 {dist_actual/pulses*1e6:.3f} 微米')
print(f'理论：每脉冲对应 {(2*3.14159*0.055)/(8192*19)*1e6:.3f} 微米')
"
```

**常见误差来源**：

| 误差来源 | 典型量级 | 解决方案 |
|----------|---------|---------|
| 轮径测量不准 | 0.5%~2% | 直接标定替代理论值 |
| 轮子打滑 | 0~5% | 加大摩擦或降低加速度 |
| 地面不平 | 视情况 | 结合外部传感器修正 |
| 编码器量化误差 | ≪0.1% | 可忽略 |

### 3.2 转向电机（GM6020）：角度标定（零位偏移）

GM6020 编码器上电后的初始值并不等于"正前方朝向"，需要标定**零位偏移（steer_offset）**：

**标定步骤**：

```bash
# 1. 手动将所有舵轮调整到正前方（与底盘前进方向一致）
# 2. 读取此时每个 GM6020 的编码器角度
ros2 topic echo /dji_motor_states
# 记录 DJI6020_1/2/3/4 的 angle 字段值

# 3. 将读取到的角度填入配置文件
# config/chassis_params.yaml:
#   fl_steer_offset: 330.8   ← 左前轮正前方时的编码器角度
#   fr_steer_offset:  89.6   ← 右前轮正前方时的编码器角度
#   rl_steer_offset: 207.1   ← 左后轮正前方时的编码器角度
#   rr_steer_offset:  33.0   ← 右后轮正前方时的编码器角度
```

**坐标变换链**（`docs/底盘控制节点数据流.md` 中也有说明）：

```
编码器角度 (encoder_angle, 0-360°)
      │
      │  - steer_offset   （减去零位偏移）
      ▼
机械角度 (mechanical_angle, 0° = 正前方)
      │
      │  × steer_direction  （修正安装方向）
      ▼
运动学角度 (kinematic_angle, 逆解算使用)
```

## 4. 底盘里程计（Odometry）

### 4.1 从轮子速度推算底盘速度

**舵轮底盘正运动学**（SteerWheelKinematics::forwardKinematics）：

```
已知：4 个舵轮各自的线速度 v_i（m/s）和转向角 θ_i（°）
求：底盘速度 (vx, vy, ωz)

每个轮子贡献：
  v_ix = v_i × cos(θ_i)
  v_iy = v_i × sin(θ_i)

最小二乘融合：
  vx = 平均(v_ix)
  vy = 平均(v_iy)
  ωz = 由左右速度差估算
```

**代码链路**（`chassis_control_node.cpp:285-300`）：

```cpp
// 从电机反馈获取实际轮子状态
// 完整换算：RPM → 输出轴RPM（÷减速比19）→ 线速度（× 轮周长 / 60s）
// velocity (m/s) = (rpm / gear_ratio / 60) × (2π × wheel_radius)
//                = rpm × (2π × 0.055) / (19 × 60)
fl_actual.velocity = motor_states_[motor_names_.fl_drive].rpm
                     * (2.0 * M_PI * 0.055) / (19.0 * 60.0);  // RPM→m/s

// 正运动学还原底盘速度
kinematics_->forwardKinematics(
    fl_actual, fr_actual, rl_actual, rr_actual,
    msg.linear_x, msg.linear_y, msg.angular_z
);
```

### 4.2 底盘位置积分（基础里程计）

若需要追踪底盘位置（x, y, θ），需在控制循环中对速度积分：

```python
# 伪代码：里程计积分（控制频率 100 Hz，dt = 0.01 s）
dt = 0.01  # 秒

x     += (vx * cos(theta) - vy * sin(theta)) * dt
y     += (vx * sin(theta) + vy * cos(theta)) * dt
theta += wz * dt
```

> **注意**：纯轮式里程计会随时间漂移。高精度场景需融合 IMU（惯性测量单元）或激光雷达 SLAM。

## 5. 给定目标点的位置控制策略

### 5.1 单轮位置控制（舵角定位）

对于 GM6020 转向电机，实现"旋转到目标角度"的核心是**串级位置 PID 控制**：

```
目标角度 (度)
    │
    ▼
[位置环 PID]  ────  比较目标角度与当前角度，输出目标转速（RPM）
    │                  处理过零问题：选择最短旋转路径（-180°~+180°）
    ▼
[速度环 PID]  ────  比较目标转速与实际转速，输出电压值
    │
    ▼
GM6020 电机电压  ────  电机实际旋转
    │
    ▼ （反馈）
编码器读取实际角度（0-360°）
```

### 5.2 底盘点对点运动控制

要让底盘从当前位置运动到指定目标点 (x_goal, y_goal)，推荐以下分层策略：

#### 方法一：速度规划 + 里程计反馈（最常用）

```
1. 读取当前位置：(x_cur, y_cur, θ_cur)（来自里程计或外部定位）
2. 计算位移向量：
     dx = x_goal - x_cur
     dy = y_goal - y_cur
     dist = sqrt(dx² + dy²)   ← 剩余距离
     angle_to_goal = atan2(dy, dx)  ← 目标方向
3. 底盘姿态控制（角速度）：
     angle_error = angle_to_goal - θ_cur（归一化到 ±180°）
     ωz = Kp_yaw × angle_error
4. 前进速度（接近目标时减速）：
     vx = min(Kp_dist × dist, v_max)
5. 发布 /cmd_vel（geometry_msgs/Twist）
6. 到达判断：if dist < 0.05m：停止
```

#### 方法二：直接发布 /cmd_vel（外部位置控制器）

本项目通过 `/cmd_vel` 话题接收底盘速度指令（`chassis_control_node.cpp:64-67`）：

```cpp
cmd_vel_sub_ = this->create_subscription<geometry_msgs::msg::Twist>(
    "/cmd_vel", 10,
    std::bind(&ChassisControlNode::cmdVelCallback, this, std::placeholders::_1)
);
```

外部位置控制器（如导航框架 Nav2、或自定义节点）可以通过发布 `/cmd_vel` 驱动底盘：

```python
# ROS2 Python：发布底盘速度命令示例
import rclpy
from geometry_msgs.msg import Twist

rclpy.init()
node = rclpy.create_node('position_controller')
pub  = node.create_publisher(Twist, '/cmd_vel', 10)

msg = Twist()
msg.linear.x  = 0.5   # 前进 0.5 m/s
msg.linear.y  = 0.0   # 无横向速度
msg.angular.z = 0.0   # 无旋转
pub.publish(msg)
```

#### 方法三：舵轮全向运动到目标点（全向底盘优势）

舵轮底盘支持全向移动（vx、vy、ωz 同时独立控制），可以：
- **直接横向平移**到目标点，无需先旋转底盘朝向
- **同时旋转底盘**（保持朝向固定的前提下平移）

```cpp

    cmd_vx_, cmd_vy_, cmd_wz_,
    fl_cmd, fr_cmd, rl_cmd, rr_cmd
```

## 6. 串级 PID 控制实现（代码详解）

### 6.1 PID 控制器结构

本项目实现了带死区、积分限幅和输出限幅的通用 PID 控制器（`include/motor_control_ros2/pid_controller.hpp`）：

```cpp
double PIDController::calculate(double target, double feedback) {
    // 1. 计算原始误差
    double raw_error = target - feedback;
    
    // 2. 死区处理（避免微小震荡）
    if (std::abs(raw_error) < params_.dead_zone) {
        error_[0] = 0.0;
    } else {
        error_[0] = raw_error;
    }
    
    // 3. P 项：比例控制
    double p_out = params_.kp * error_[0];
    
    // 4. I 项：积分控制（含限幅，防止积分饱和）
    i_out_ += params_.ki * error_[0];
    i_out_  = std::clamp(i_out_, -params_.i_max, params_.i_max);
    
    // 5. D 项：微分控制（差分，响应误差变化速率）
    // 注意：此实现假设**固定控制频率**（默认 200 Hz），kd 已将 1/dt 吸收。
    // 若控制频率变化，需将 kd 除以实际 dt 以保持一致行为。
    double d_out = params_.kd * (error_[0] - error_[1]);
    
    // 6. 保存上次误差
    error_[1] = error_[0];
    
    // 7. 总输出（含限幅）
    double output = p_out + i_out_ + d_out;
    return std::clamp(output, -params_.out_max, params_.out_max);
}
```

### 6.2 串级控制器（位置→速度→输出）

`cascade_controller.hpp` 实现了三种模式（`POSITION` / `VELOCITY` / `DIRECT`）：

```cpp
case ControlMode::POSITION: {
    // 处理角度过零：选择最短旋转路径
    double angle_error = position_target - position_feedback;
    
    // 归一化到 [-180°, +180°]，选择最短路径
    // 此 while 循环实现等价于：
    //   angle_error = fmod(angle_error + 180.0, 360.0) - 180.0;
    // 当误差始终在 ±360° 以内（正常控制情况）时，两种写法最多循环 1 次。
    while (angle_error >  180.0) angle_error -= 360.0;
    while (angle_error < -180.0) angle_error += 360.0;
    
    // 位置环 → 输出速度目标（RPM）
    double normalized_target   = position_feedback + angle_error;
    double vel_target_from_pos = position_pid_.calculate(
        normalized_target, position_feedback);
    
    // 速度环 → 最终电压/电流输出
    return velocity_pid_.calculate(
        vel_target_from_pos, velocity_feedback);
}
```

### 6.3 完整调用链（DJI 电机）

```
DJIMotor::updateController()                   ← 200Hz 控制循环调用
    │
    ├─ position_target_（度）     ← setPositionTarget() 设置
    ├─ getAngleDegrees()          ← 编码器反馈（度）
    ├─ raw_rpm_                   ← 电机速度反馈（RPM）
    │
    └─ cascade_controller_.update(...)
           │
           ├─ 位置 PID：计算目标 RPM
           └─ 速度 PID：计算输出电压
                   │
                   └─ setOutput(int16_t)  → CAN 帧发送
```

**代码位置** `dji_motor.cpp:164-176`：

```cpp
void DJIMotor::updateController() {
    double output = cascade_controller_.update(
        position_target_,                    // 位置目标（度）
        velocity_target_,                    // 速度目标（RPM）
        static_cast<double>(target_output_), // 直接输出（DIRECT 模式）
        getAngleDegrees(),                   // 位置反馈（度，0-360）
        static_cast<double>(raw_rpm_)        // 速度反馈（RPM）
    );
    setOutput(static_cast<int16_t>(output));
}
```

### 6.4 PID 参数配置（YAML）

PID 参数通过 `config/pid_params.yaml` 配置，无需重新编译（`pid_params.yaml:6-45`）：

```yaml
dji_motors:
  GM6020:
    position_pid:
      kp: 20.0          # 位置比例系数
      ki: 1.0           # 位置积分系数
      kd: 0.0           # 位置微分系数
      i_max: 50.0       # 积分限幅（输出为 RPM，故限制为 50 RPM）
      out_max: 700.0    # 位置环最大输出（目标转速上限，RPM）
      dead_zone: 0.1    # 位置死区（0.1° 内不响应）
    velocity_pid:
      kp: 40.0
      ki: 2.0
      kd: 0.1
      i_max: 1500.0
      out_max: 16384.0  # GM6020 最大电压
      dead_zone: 1.0    # 速度死区（1 RPM 内不响应）

  GM3508:
    position_pid:
      kp: 20.0
      out_max: 1000.0   # 驱动轮位置环输出上限（RPM）
      dead_zone: 0.5    # 0.5° 死区
    velocity_pid:
      kp: 8.0
      ki: 1.0
      kd: 0.5
      out_max: 16384.0  # GM3508 最大电流
      dead_zone: 5.0    # 5 RPM 死区
```

## 7. PID 调试方法与常见问题

### 7.1 推荐调试顺序

**先调内环（速度），再调外环（位置）**：

```
步骤 1：单独调速度环
─────────────────────────────────────────────────────
设置：mode = VELOCITY，给定固定目标转速（如 300 RPM）

调参顺序：
  1. ki=0, kd=0，从小 kp 开始（如 kp=5），逐渐增大
  2. 观察目标响应曲线：
     - kp 偏小 → 响应迟缓，速度跟不上
     - kp 适中 → 快速接近目标
     - kp 过大 → 速度震荡
  3. 如有稳态误差（停在目标±10RPM 以外），适量增加 ki（如 0.5~2.0）
  4. 如有高频震荡，增加 kd（如 0.1~0.5）

步骤 2：在速度环稳定后调位置环
─────────────────────────────────────────────────────
设置：mode = POSITION，给定固定目标角度（如 90°）

调参顺序：
  1. ki=0, kd=0，从小 kp 开始（如 kp=5~10）
  2. 增大 kp 直到旋转速度满足需求，注意 out_max 限制了最大速度
  3. 如有超调（越过目标角度后回摆），增加 kd 或减小 kp
  4. 如停在目标 ±1° 以外，增加 ki（注意 i_max 限幅）
```

### 7.2 常见问题与解决方案

#### 问题 1：电机到达目标位置后持续震荡

**原因**：kp 过大，死区设置过小，或 ki 过大导致积分饱和

**解决方案**：

```yaml
# 适当增大死区
position_pid:
  dead_zone: 0.5    # 从 0.1 增大到 0.5°（可接受范围内）
  kd: 2.0           # 增加微分项阻尼

velocity_pid:
  dead_zone: 5.0    # 速度也设置合理死区
```

#### 问题 2：位置环有稳态误差（停在目标附近但不精确）

**原因**：ki 为 0 或 i_max 设置过小，积分项无法补偿静摩擦力

**解决方案**：

```yaml
position_pid:
  ki: 0.5         # 添加适量积分
  i_max: 30.0     # 确保积分上限足够
```

#### 问题 3：过零跳变（舵轮从 359° 跳变到 1° 时反转）

**原因**：角度误差未归一化，直接计算导致误差巨大（如 -358°）

**本项目解决方案**（`cascade_controller.hpp:102-106`）：

```cpp
// 归一化角度误差到 [-180°, +180°]，自动选择最短旋转路径
double angle_error = position_target - position_feedback;
while (angle_error >  180.0) angle_error -= 360.0;
while (angle_error < -180.0) angle_error += 360.0;
// 等价的紧凑写法（当误差在 ±360° 以内时等价）：
// angle_error = fmod(angle_error + 180.0, 360.0) - 180.0;
```

#### 问题 4：GM3508 累积编码器跳变（从一圈跳到下一圈时位置突变）

**原因**：过零检测阈值不合理，或控制频率太低导致每帧编码器变化超过半圈（4096 脉冲）

**定量分析**：
```
过零检测失效的临界转速：
  在控制周期 dt = 1/200s = 5ms 内，电机轴转过半圈（4096 脉冲）时失效。
  临界转速 = (8192 脉冲/圈 / 2) / (5ms / 1000) = 4096 / 0.005 = 819200 脉冲/s
           = 819200 / 8192 圈/s × 60 ≈ 6000 RPM

  GM3508 额定最高转速约 480 RPM（空载），远低于临界值，
  因此在正常工作频率（200 Hz）下过零检测始终可靠。
```

**解决方案**：
- 保持控制循环频率 ≥ 200 Hz（默认设置）
- 若使用其他电机或极高转速，按上述公式验证安全裕量

#### 问题 5：到达目标点后底盘缓慢漂移

**原因**：里程计积分误差累积，底盘认为"已到达"但实际位置有偏差

**解决方案**：
- 增加外部绝对定位（激光雷达、视觉 AprilTag）
- 使用 ROS2 Nav2 导航框架，融合多种传感器

#### 问题 6：切换控制模式后输出突变

**原因**：积分项在切换前已累积大量值

**本项目解决方案**（`cascade_controller.hpp:38-43`）：

```cpp
void setMode(ControlMode mode) {
    if (mode_ != mode) {
        // 模式切换时重置所有 PID 状态（清除积分和误差历史）
        position_pid_.reset();
        velocity_pid_.reset();
        mode_ = mode;
    }
}
```

## 8. 整体控制流程图

```
外部目标点 (x_goal, y_goal, θ_goal)
          │
          ▼
┌─────────────────────────┐
│  上层位置控制器          │   可用 Nav2 / 自定义节点
│  （里程计反馈 + PID）     │
│  输出: vx, vy, wz        │
└────────────┬────────────┘
             │ /cmd_vel (Twist)
             ▼
┌─────────────────────────┐
│  chassis_control_node    │   chassis_params.yaml
│  （100 Hz 控制循环）     │
│  ① 逆运动学解算          │   4×(舵角°, 驱动速度 m/s)
│  ② 舵角最短路径优化       │
│  ③ 机械角→编码器角换算    │
└────────────┬────────────┘
             │ /dji_motor_command_advanced
             │ (8 条消息: 4×转向位置目标 + 4×驱动速度目标)
             ▼
┌─────────────────────────┐
│  motor_control_node      │   motors.yaml + pid_params.yaml
│  （200 Hz 控制循环）     │
│  ┌──GM6020（转向）──┐   │
│  │  位置 PID (度)   │   │   角度反馈: 编码器 0-8191 → 0-360°
│  │  速度 PID (RPM)  │   │   输出: 电压 ±30000
│  └──────────────────┘   │
│  ┌──GM3508（驱动）──┐   │
│  │  速度 PID (RPM)  │   │   转速反馈: RPM（含过零计数）
│  │  电流输出         │   │   输出: 电流 ±16384
│  └──────────────────┘   │
└────────────┬────────────┘
             │ CAN 总线（921600 bps）
             ▼
┌─────────────────────────┐
│  DJI 电机硬件            │   物理执行
│  GM6020 × 4（转向）      │
│  GM3508 × 4（驱动）      │
└────────────┬────────────┘
             │ CAN 反馈（~1ms 周期）
             │  raw_angle (0-8191)
             │  raw_rpm   (int16)
             └──────────────────────► 编码器反馈回控制闭环
```
